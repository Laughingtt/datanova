## 1. 2025-2026年中国智能问数专利全景

### 1.1 专利爆发态势

2025-2026年中国智能问数/ChatBI领域专利呈现爆发式增长，核心驱动因素：

- **大模型普及**：2025年生成式AI用户突破6.02亿，普及率42.8%
- **政策推动**：工信部"数据要素×"三年行动计划（2024-2026），7个试点城市建成数据标注基地
- **审查加速**：CNIPA启动大模型专利池建设，2025-2026累计优先审查约60.4万件、快速审查约11.6万件
- **审查新规**：2026年1月1日起实施的《专利审查指南》修订稿明确AI发明专利审查标准

### 4.2 突破口1：反馈驱动的多信号融合检索闭环（🥇最高优先）

**为什么是专利空白区？**

调研发现：
- 中国现有专利（CN119721255A）仅覆盖"执行结果反馈到Agent上下文"的简单闭环
- 华为云CN120045576A仅覆盖"历史SQL候选集匹配"的静态方案
- RLHF方向（学术）需要GPU微调，与DataNova的轻量级方案路线不同
- **无人专利化"多信号融合评分+非对称反馈加权+反馈门控过滤+懒同步刷新"的组合方案**

**DataNova独有技术方案**：

```
用户👍👎反馈 → feedback_category分类
    ↓
反馈聚合（getFeedbackStatsBySQL）: 正/负反馈计数 + 高频反馈类别
    ↓
反馈门控过滤: 仅保留 verified OR success≥3；排除 negative≥3 AND positive=0
    ↓
多信号复合评分:
  - 关键词重叠度（+2/question关键词, +1/table关键词）
  - 验证状态加成（+3/已验证）
  - 执行历史可靠性（+min(successCount,5), -3/错误>成功）
  - 非对称反馈加权（+1/正反馈上限+3, -2/负反馈上限-10）
    ↓
懒同步刷新: lookup_examples时自动upsert + ON CONFLICT DO UPDATE
    ↓
检索排序 → few-shot示例注入Agent → SQL生成优化
```

**差异化要素（相对CN119721255A和华为云CN120045576A）**：
1. **四源信号融合评分**（关键词+验证+执行历史+反馈）— 仅有专利覆盖"简单反馈到上下文"
2. **非对称反馈加权**（负反馈权重2×正反馈）— 无专利覆盖
3. **反馈门控硬过滤**（verified/success阈值+negative排除）— 无专利覆盖
4. **懒同步刷新机制**（查询时upsert+自动难度分类）— 无专利覆盖

### 4.3 突破口2：双路径SQL生成的路径调度与回退机制（🥈次高优先）

**专利环境分析**：

CN119917526A和CN119721233B已覆盖"语义层→确定性SQL"核心思路，但：
- 这两项专利均为**单路径确定性方案**（仅语义层路径）
- **未覆盖"语义层确定性路径+AI回退路径"的双路径调度**
- **未覆盖"语义层未命中时的自动回退策略"**
- **未覆盖"路径选择信号和回退触发机制"**

**DataNova独有技术方案**：

```
自然语言查询
    ↓
lookup_semantic_layer: 三层搜索（指标→维度→模型）
    ↓
    ├── 命中 → buildSemanticSql() 生成确定性SQL + 返回模型/维度/时间元信息
    │          → 模型判断: 直接执行 or 调整时间/维度后执行
    │          + skip_probe=true 标记
    │          + /* source: semantic_layer */ 来源标记
    │
    └── 未命中 → Agent回退到Schema探索路径
               discover_schema → execute_sql
               + /* source: agent_exploration */ 来源标记
    ↓
SQL安全验证管道 → 执行 → 结果+来源标记
    ↓
用户反馈 → 更新反馈库 → 影响后续路径选择偏好
```

**差异化要素（相对CN119917526A/CN119721233B）**：
1. **双路径调度机制**：语义层优先+AI回退，而非单一确定性路径
2. **三层语义搜索的回退触发**：指标→维度→模型逐层搜索，全部未命中才回退
3. **skip_probe标记**：明确告知Agent跳过探测，减少无效工具调用
4. **来源标记系统**：`/* source: semantic_layer */` vs `/* source: agent_exploration */`，支持结果溯源和路径统计
5. **反馈影响路径偏好**：正向反馈多的路径权重更高

### 4.4 突破口3：查询自纠链路追踪与意图感知检索（🥉第三优先）

**专利环境**：**完全空白区** — 现有专利仅记录离散SQL执行事件，无人建模查询间的纠错关系。

**DataNova独有技术方案**：

```sql
-- sql_query_history表中的关键字段
parent_query_id    -- 纠正SQL链接到失败前驱（DAG结构）
correction_round   -- 0=原始, 1+=纠正轮次
intent_type        -- new_query/refine/drill_down/compare/explain/correction
```

**差异化要素**：
1. **DAG纠错链路**：parent_query_id构建有向无环图，追踪从失败到成功的完整路径
2. **意图分类感知检索**：基于intent_type在匹配意图的示例子集中检索
3. **纠错学习**：使用DAG终端成功节点作为few-shot示例，惩罚失败起点

## 5. 可申请专利的具体方案

### 5.1 专利1（P0）：一种反馈驱动的多信号融合查询示例检索与SQL生成闭环方法

**申请类型**：发明专利  
**预估授权可能性**：高（专利空白区+完整技术方案+可实现性）

**权利要求书框架**：

```
1. 一种反馈驱动的多信号融合查询示例检索与SQL生成闭环方法，其特征在于，包括：
   S1. 全量记录每次SQL查询的执行信息至查询历史库，包含执行状态、耗时、返回行数、
       错误信息，并通过parent_query_id和correction_round构建查询纠错DAG；
   S2. 采集用户对查询结果的结构化反馈数据，包含反馈类型（正向/负向）和反馈分类
       （准确性/完整性/可理解性等）；
   S3. 基于查询历史和反馈数据执行反馈门控过滤：仅保留已验证或成功执行超过阈值的
       示例，排除负反馈超过阈值且无正向反馈的示例；
   S4. 对候选示例执行多信号复合评分，所述评分融合以下信号：
       a) 关键词重叠度信号；
       b) 验证状态信号；
       c) 执行历史可靠性信号；
       d) 反馈加权信号，其中负反馈权重为正反馈权重的2倍以上；
   S5. 基于复合评分排序检索结果，将排序靠前的示例作为few-shot示例注入Agent上下文；
   S6. Agent基于few-shot示例生成SQL查询并执行。

2. 根据权利要求1所述的方法，其特征在于，所述反馈门控过滤包括：
   第一层：保留is_verified=true或success_count≥3的示例；
   第二层：排除negative_count≥3且positive_count=0的示例；
   所述两层过滤在检索前执行，减少无效候选示例。

3. 根据权利要求1所述的方法，其特征在于，所述反馈加权信号的计算包括：
   正向反馈贡献：+1/次，上限+3；
   负向反馈贡献：-2/次，上限-10；
   负向反馈权重为正向反馈的2倍，体现保守纠错策略。

4. 根据权利要求1所述的方法，其特征在于，还包括查询时懒同步刷新：
```

## 结论：DataNova创新突破口总结

### 🏆 最佳突破口：反馈驱动的多信号融合检索闭环

**理由**：
1. **专利空白度最高**：中国2025-2026年无人专利化此方案
2. **技术壁垒最强**：四信号融合+非对称加权+门控过滤的组合难以绕过
3. **商业价值最大**：直接解决智能问数核心痛点——SQL准确率持续提升
4. **规避风险最低**：与现有专利（CN119721255A、华为云CN120045576A）差异明确

### 🥈 次佳突破口：双路径SQL生成的路径调度与回退机制

**理由**：
1. **关键差异化**：CN119917526A仅覆盖单路径确定性SQL，DataNova的双路径调度是增量创新
2. **规避策略清晰**：强调"回退触发+来源标记+路径偏好+skip_probe"4个差异化要素
3. **但风险较高**：CN119917526A的核心权利要求可能较宽，需详细分析

### 🥉 第三突破口：查询自纠链路追踪与意图感知检索

**理由**：
1. **完全空白区**：无任何现有专利覆盖查询纠错DAG结构
2. **但壁垒较低**：parent_query_id+correction_round的技术方案相对简单
3. **建议作为补充专利**：与专利1（反馈闭环）组合申请，增强保护范围
